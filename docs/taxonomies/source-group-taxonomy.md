# 公众号分组体系

这份文档用于约束 FocusLedger 中公众号来源的主分组体系。

## 设计原则

- 分组只承担“主归属”职责，一个公众号通常只保留一个主分组
- 分组使用层级路径，格式统一为 `一级/二级/三级`
- 分组优先描述“来源是什么机构、扮演什么角色”，而不是描述单篇文章主题
- 横向补充信息交给来源标签，不在分组里解决所有问题
- 控制总深度，避免出现过长、过细、难以维护的树状结构

## 判定顺序

1. 先判断来源身份：媒体、研究机构、公司、官方机构、学术、投资机构等
2. 再判断组织类型：卖方、买方、咨询、协会、监管、创始人 IP 等
3. 最后才判断长期稳定覆盖方向：宏观、策略、行业、科技、消费等
4. 如果一个号跨多个方向输出很多内容，也仍然只保留一个最稳定的主分组，其余交给来源标签

## 常见映射建议

- 券商研究团队优先进入 `研究/卖方/*`
- 公募、私募、资管团队优先进入 `投资/基金与资管` 或 `研究/买方/*`
- 政府部门、监管机关、政策发布号优先进入 `机构/政府与监管`
- 行业协会、智库、研究会优先进入 `机构/协会与智库`
- 企业官方公众号优先进入 `公司/官方发布`
- 个人色彩很强但长期围绕公司、创业、产业经营输出的号，可进入 `公司/创始人IP`
- 新闻聚合、编辑部、媒体品牌号优先进入 `资讯/*`

## 维护建议

- 先保证稳定，再追求细分；新建分组必须能服务多个来源，而不是只服务一个特例
- 当某个分组长期只容纳 1 个来源时，优先考虑是否应该降级为标签
- 如果一个新增来源很难归入现有分组，优先修改本文件，再执行 AI 分组
