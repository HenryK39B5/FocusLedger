# 公众号标签体系

这份文档用于约束 FocusLedger 中公众号来源标签。

## 设计原则

- 来源标签描述“这个公众号长期是什么、长期关注什么、长期怎么输出”
- 来源标签允许多标签共存，但应尽量稳定，不追逐短期热点
- 采用多级路径，便于后续筛选父标签时覆盖子标签
- 分组负责主归属，标签负责横向切片，两者不要重复承担同一职责
- 同一公众号建议控制在 3 到 6 个来源标签

## 标签维度说明

- `身份/*`：描述来源角色
- `覆盖/*`：描述长期研究或内容覆盖范围
- `风格/*`：描述稳定的输出方式
- `区域/*`：描述长期关注的主要市场区域

## 标注建议

- 一个公众号至少应有 1 个 `身份/*` 标签
- 长期研究型公众号通常还应有 1 到 3 个 `覆盖/*` 标签
- 如果某个号输出方式非常稳定，再补 `风格/*`
- 只有当来源长期明确覆盖某个市场区域时，才补 `区域/*`
- 不要把短期热门主题写进来源标签，例如某号偶尔写一次 AI，不代表要打 `覆盖/AI`

## 示例

- 某券商宏观团队：`身份/卖方`、`覆盖/宏观`、`风格/深度研究`
- 某科技媒体：`身份/媒体`、`覆盖/科技`、`覆盖/AI`、`风格/快讯`
- 某企业创始人账号：`身份/创始人IP`、`覆盖/公司研究`、`风格/评论观点`
