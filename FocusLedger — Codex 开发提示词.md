# FocusLedger — Codex 开发提示词

你是我的 coding agent。请在当前空文件夹中，从零开始初始化并开发一个本地可运行的中型项目，项目名称为 **FocusLedger**。

这是一个**面向中文财经信息流的个性化注意力管理系统**。  
它的核心不是“做一个普通资讯聚合器”，而是：

- 从用户精挑细选的公众号来源中获取财经文章
- 将文章结构化理解
- 识别“增量信息”与“重复共识”
- 基于用户长期偏好与多轮反馈，生成个性化精选流
- 帮助用户管理有限注意力，把时间花在最值得看的少数内容上

当前阶段仅开发**本地 MVP / 本地可演示版本**。  
暂时**不要**实现 OpenClaw、飞书、微信消息接入层。  
但是系统架构必须预留后续接入能力。

---

## 一、产品定位

请把产品理解为：

> 一个运行在本地的、面向中文财经公众号信息流的个性化注意力管理与增量识别系统。

它服务的不是“全网内容抓取”，而是：

- 用户已经有一批自己信任的公众号来源
- 用户希望系统每天自动获取这些来源的新内容
- 系统帮用户从中挑出最值得读的几篇
- 系统告诉用户：这篇有什么新东西，为什么值得看
- 用户可以通过反馈不断校准推荐结果

请注意，这个产品未来可以扩展到：
- 卖方研报
- 财经新闻
- 深度文章
- 行业观察
- 市场评论

但当前首要数据源是：**用户关注的微信公众号文章**

---

## 二、核心产品问题

请围绕下面 5 个问题设计和实现系统：

1. 今天抓到的公众号财经文章里，哪几篇最值得读？
2. 某篇文章相比过去同主题内容，到底“新”在哪里？
3. 用户真正长期关注哪些主题、风格和信息密度？
4. 用户的反馈如何持续更新推荐逻辑？
5. 如何把这个系统做成未来可接 OpenClaw / 飞书 / 微信的“核心引擎”？

## 六、产品核心能力

请把本地 MVP 的核心能力做完整，不要只停留在“抓取 + 摘要”。

### 1. 公众号文章采集

支持至少这些逻辑能力：

- 配置需要追踪的公众号来源
- 获取文章列表
- 获取文章正文
- 获取可得的互动数据
- 原始 HTML 与解析结果分离存储
- 支持定时任务触发抓取
- 支持手动触发抓取

### 2. 文章结构化理解

对每篇文章抽取和存储：

- 标题
- 来源公众号
- 发布时间
- 作者（若可得）
- 原文链接
- 正文文本
- 摘要
- 主题标签
- 实体标签（公司、行业、资产、机构等）
- 内容类型（深度、快评、行业观察、市场复盘、策略观点等）
- 核心结论
- 关键变量
- 催化因素
- 风险点
- 风格标签
- 推荐理由
- 是否适合当前用户

### 3. 历史相似内容召回

必须支持：

- 基于主题和向量召回历史相似文章
- 限定时间窗口（如近 7 天 / 30 天 / 90 天）
- 同一主题下进行相似内容比较

### 4. 增量信息识别

这是重点。

对于一篇新文章，请输出：

- 重复共识内容
- 真正增量内容
- 增量类型：
  - 新事实
  - 新框架
  - 新比较
  - 新判断
  - 新催化
- 一个可解释的增量分数
- 一段中文解释：为什么这篇值得看

### 5. 个性化推荐

为用户维护偏好画像：

- 关注主题
- 不关注主题
- 偏好内容类型
- 偏好风格
- 已保存内容
- 反馈历史
- 跟踪主题

并基于这些信息做推荐排序。

### 6. 多轮反馈校准

支持以下反馈动作：

- 多推类似内容
- 少推类似内容
- 关注这个主题
- 不感兴趣
- 收藏
- 标记已读
- 加入我的关注清单

反馈必须真实落库，并影响后续排序。
